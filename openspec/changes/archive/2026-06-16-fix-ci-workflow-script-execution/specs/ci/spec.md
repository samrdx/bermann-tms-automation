# CI Specification

## Purpose

Define mandatory behavior for GitHub Actions workflows so CI status truthfully reflects project health and keeps report publication reliable.

## Requirements

### Requirement: Required Script Execution by Trigger

The CI workflows MUST execute the required `npm run` scripts for each trigger type and selected environment.

#### Scenario: Manual dispatch runs only the selected environment

- GIVEN a user starts `workflow_dispatch` and selects `QA`
- WHEN the workflow executes test jobs
- THEN required QA scripts MUST run
- AND DEMO-only scripts MUST NOT run in that dispatch

#### Scenario: Push or scheduled runs execute all mandatory environments

- GIVEN a `push` to an allowed branch or a nightly schedule trigger
- WHEN CI starts environment test jobs
- THEN the workflow MUST execute required scripts for both QA and DEMO where defined
- AND each environment result MUST be recorded independently

### Requirement: Workflow to Package Script Contract

The CI pipeline MUST fail fast when a workflow references a missing `npm` script.

#### Scenario: Workflow references valid scripts

- GIVEN workflow files contain `npm run <script>` commands
- WHEN the CI integrity guard validates those commands against `package.json`
- THEN validation MUST pass when every script exists

#### Scenario: Workflow references an undefined script

- GIVEN a workflow references `npm run` for a script not present in `package.json`
- WHEN the CI integrity guard executes
- THEN the guard MUST fail the job before running test suites
- AND the failure output MUST identify the missing script name

### Requirement: Environment Aligned Artifact Paths

Workflow artifact upload steps SHALL use the same output paths generated by Playwright reporters for each environment.

#### Scenario: QA and DEMO reports are uploaded from correct directories

- GIVEN Playwright writes reports using environment scoped directories
- WHEN upload artifact steps run
- THEN QA artifacts MUST be collected from QA report directories
- AND DEMO artifacts MUST be collected from DEMO report directories

#### Scenario: Report directory mismatch is detected

- GIVEN a workflow upload step points to a directory that is not generated by the configured reporter
- WHEN artifact upload executes
- THEN CI MUST surface the mismatch in logs
- AND final status MUST indicate failure when mandatory artifacts are missing

### Requirement: Explicit Final Fail Gate

Mandatory test suites MAY run with `continue-on-error` to preserve report generation, but workflows MUST enforce a final fail gate based on suite outcomes.

#### Scenario: Mandatory suites pass and workflow succeeds

- GIVEN mandatory suites complete without failure
- WHEN the final gate evaluates suite outcomes
- THEN the job MUST finish with success
- AND report generation and publishing steps SHALL complete

#### Scenario: At least one mandatory suite fails

- GIVEN one or more mandatory suites fail during execution
- WHEN report generation and artifact publication finish
- THEN the final gate MUST fail the job
- AND CI status MUST be red for that workflow run

### Requirement: Dispatch Environment Behavior Must Be Explicit

Manual dispatch environment behavior MUST be explicit and deterministic.

#### Scenario: Default dispatch behavior is defined

- GIVEN a user opens workflow manual run options
- WHEN no custom environment strategy is provided
- THEN the workflow MUST define a clear default environment behavior
- AND the workflow file SHOULD document this behavior inline

#### Scenario: Unsupported environment input

- GIVEN a dispatch input value outside supported options
- WHEN the workflow evaluates environment selection
- THEN the workflow MUST fail validation before test execution
- AND the error message MUST state supported values
