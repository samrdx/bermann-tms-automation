# Documentation Spec

## Bermann TMS - QA Automation Framework README

### Overview
The `README.md` acts as the primary user-facing documentation for the repository.

### Requirements

#### Project Status DASHBOARD
- The `README.md` MUST display a "Project Status Dashboard" table as seen in `GEMINI.md`.

#### Allure Reports Documentation
- The `README.md` MUST include clear instructions for Allure reporting commands for both QA and Demo environments.

#### Demo Environment Support
- The `README.md` MUST list the `test:demo:*` scripts from `package.json`.

#### Test Classification
- The `README.md` MUST define the difference between **Atomic** (independent) and **Legacy** (sequential) tests.

#### Environment Consistency
- The `README.md` MUST use `TMS_USERNAME` and `TMS_PASSWORD` as the primary credential variables.
